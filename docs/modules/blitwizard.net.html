Module blitwizard.net

The blitwizard net namespace contains networking functionality.

If you want to create a multiplayer game, you may want to have a look at it.

Functions

server_close (port)
Close the server which was opened at the given port. This will also close down all connections you accepted through it.

Parameters:

  • port number the port of the server listener to be closed
server_open (port, accept_callback)
Open up a server listener at the given port number. This will allow you to get incoming connections at the given port, accept them and then exchange data with them. A server can receive an arbitrary amount of client connections at once.

Parameters:

  • port number the port to open a server at
  • accept_callback function this function will be called every time you get a new connection. The first parameter is the listener port number, the second parameter the new connection. You may want to use connection:set on each new connection to set a read callback so you can receive data from it.

Class connection

Represents a network connection.
connection.settings
This is how you should submit settings to connection:new. (THIS TABLE DOESN'T EXIST, it is just a guide on how to construct it yourself with concrete settings values. See connection:new's usage example!)

Fields:

  • server string the target server's host name or ip address
  • port number the target server's TCP/IP port you want to connect to
  • linebuffered boolean (optional) whether you want the read callback to collect full lines up to the next line break and then return you those complete lines (=linebuffered set to true), or to pass on whatever it receives instantly no matter whether it is just a partial line or not (=linebuffered set to false)
  • lowdelay boolean (optional) this allows you to enable low delay sending.

    If disabled (the default), sending on a connection will sometimes be delayed (for a few milliseconds) to assemble more data in a bigger packet. This notably increases connection throughput (=you can send more data in the same amount of time), but it also increases connection latency (=the amount of time delay between sending out a specific data item and the target receiving it).

    If enabled, you will get degraded data throughput but smaller latency. This is only recommended if the data is extremely time-critical, e.g. player positions in a fast-paced action game.

connection:close ()
Close the given connection. You will no longer be able to send things on it.
connection:new (settings, open_callback, read_callback, close_callback)
Create a new network connection to the given target.

Parameters:

  • settings table a table with the available settings in it you want to choose
  • open_callback function a callback function that will be called when the connection has beeen successfully opened. As soon as the connection is open, you may use connection:send. The first parameter should be 'self' for the connection that was opened. (see usage example below)
  • read_callback function a callback function that will be called when the connection has been opened at some point and data has arrived (sent by whomever you opened a connection to). The first parameter should be 'self'. As second parameter, the data (string) will be passed which arrived.
  • close_callback function a callback function that will be called when your connection was closed (or it failed to open). It will be triggered both by an explicit connection close through connection:close, and a connection closing because your connection target has closed it. The first parameter should be 'self'. (see usage example below)

Returns:

    userdata the new connection

Usage:

       -- open up a connection to google.com, request the contents
   -- and print out the response:
   my_connection = blitwizard.net.connection:new(
       {server="google.com", port=80, linebuffered=true},
   function(self)
       print("Connected! Send request:")
       -- this is a simple http request:
       self:send("GET / HTTP/1.1")
       self:send("User-Agent: blitwizard demo")
       self:send("")
   end,
   function(self, line)
       -- Received a line of data. Print it out:
       print(" >> " .. line)
   end,
   function(self)
       -- Connection was closed!
       print("Connection was closed. Goodbye!")
   end)
connection:send (data)
Send data on the connection. Please note this isn't possible before the open callback (see connection:new) has been triggered.

Parameters:

  • data string data which should be sent
